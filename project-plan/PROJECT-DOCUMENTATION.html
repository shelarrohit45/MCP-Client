1. Project Overview

MCP DevOps Client is a Python application that connects to two MCP (Model Context Protocol) servers — GitHub MCP and Email MCP — to automate repository monitoring and email notifications. No AWS or cloud hosting is required; everything runs locally on your Mac.

What the project does

  • Fetches GitHub data: open PRs, CI failures/successes, open issues, latest release
  • Sends CI failure alert emails when checks fail (last 24 hours)
  • Sends a daily digest email summarizing repo activity
  • Sends PR event emails: created, pushed, merged, rejected, approved, changes requested, CI passed/failed
  • Supports Merge/Reject buttons in PR emails via a local HTTP action server
  • Prevents duplicate CI alerts using a state file
  • Runs digest and CI checks on a schedule via APScheduler
  • Logs all activity to logs/app.log

9. Workflows Explained

CI Alert (ci_alert.py)

  1. Search GitHub for PRs with status:failure in last 24h
  2. Filter out failures already in logs/state.json
  3. Render templates/ci_alert.html
  4. Send HTML email via Email MCP
  5. Save alerted run IDs to state

Daily Digest (daily_digest.py)

  1. Fetch open PRs, open issues count, failed/successful CI (24h), latest release
  2. Render templates/digest.html (handles empty sections)
  3. Send with subject [Daily Digest] owner/repo — date

PR Events (pr_events.py)

  1. Load previous PR snapshots from logs/pr_event_state.json
  2. Compare with current PR state from GitHub
  3. Detect: created, pushed, merged, rejected, reopened, approved, changes_requested, ci_failed, ci_passed
  4. Send email for new events only (dedup via event keys)

Scheduler (scheduler.py)

  1. Uses APScheduler BlockingScheduler
  2. Job 1: run_daily_digest(send=True) — cron at digest_time
  3. Job 2: run_ci_alert() — interval every N minutes
  4. Errors are logged; scheduler keeps running

11. Troubleshooting

ProblemSolution
GitHub MCP not foundRun ./scripts/install_github_mcp.sh
Private repo returns empty dataPAT needs repo read scope
Email MCP connection failedCheck EMAIL_PASSWORD (Gmail app password)
Merge/Reject returns 403PAT needs Pull requests + Contents write
Email links don't work on phoneStart action-server; set ACTION_BASE_URL to ngrok URL
Duplicate CI alertsShould not happen — check logs/state.json
No releases in digestNormal if repo has no GitHub releases
Scheduler stoppedRe-run run-scheduler in a terminal
unhandled errors in TaskGroupUsually MCP shutdown noise; fixed for 404 releases
